TypeScript's type system goes far beyond basic annotations. Here are some patterns I reach for regularly.

## Discriminated Unions

One of TypeScript's most powerful features for modeling state:

```typescript title="types.ts"
type Result<T> =
  | { status: "success"; data: T }
  | { status: "error"; error: string }
  | { status: "loading" }

function handleResult(result: Result<string>) {
  switch (result.status) {
    case "success":
      console.log(result.data) // TypeScript knows `data` exists here
      break
    case "error":
      console.error(result.error) // TypeScript knows `error` exists here
      break
    case "loading":
      console.log("Loading...")
      break
  }
}
```

## The `satisfies` Operator

Introduced in TypeScript 4.9, `satisfies` lets you validate a type without widening it:

```typescript
type Theme = {
  colors: Record<string, string>
  spacing: Record<string, number>
}

const theme = {
  colors: {
    primary: "#2563eb",
    secondary: "#64748b",
  },
  spacing: {
    sm: 8,
    md: 16,
    lg: 24,
  },
} satisfies Theme

// theme.colors.primary is typed as "#2563eb", not string
```

## Template Literal Types

Build complex string types from simpler parts:

```typescript
type Direction = "top" | "right" | "bottom" | "left"
type Size = "sm" | "md" | "lg"
type Spacing = `${Direction}-${Size}`

// Valid: "top-sm", "right-md", "bottom-lg", etc.
function applySpacing(spacing: Spacing) {
  // ...
}
```

## The `infer` Keyword

Extract types from other types using conditional inference:

```typescript
type UnwrapPromise<T> = T extends Promise<infer U> ? U : T

type A = UnwrapPromise<Promise<string>> // string
type B = UnwrapPromise<number>          // number
```

## Const Assertions for Immutable Data

Use `as const` to get the narrowest possible type:

```typescript
const routes = {
  home: "/",
  about: "/about",
  posts: "/posts",
} as const

// typeof routes.home is "/", not string
type Route = (typeof routes)[keyof typeof routes]
// Route = "/" | "/about" | "/posts"
```

These patterns will make your TypeScript code more expressive and catch more bugs at compile time rather than at runtime.
